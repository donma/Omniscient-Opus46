API 文件

使用 Nexus REST API 建構客製化整合與自動化工作流。

API 概覽

Nexus API 採用 RESTful 架構設計，所有回應均為 JSON 格式。API 基礎 URL 為：

https://api.nexus-saas.com/v1

所有 API 請求必須透過 HTTPS 發送，支援 JSON 請求體。API 版本透過 URL 路徑管理，目前最新版本為 v1。

認證

Nexus API 使用 Bearer Token 進行認證。您可以在控制台的「設定 > API 金鑰」中生成 Token。

請妥善保管您的 API 金鑰，切勿將其暴露於前端程式碼或公開的版本控制系統中。

# 在請求標頭中加入 Authorization
curl -X GET "https://api.nexus-saas.com/v1/projects" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json"

建立專案

使用 POST 請求建立新專案：

POST /v1/projects

{
  "name": "我的新專案",
  "description": "專案描述內容",
  "visibility": "private",
  "team_id": "team_abc123"
}

回應範例

{
  "id": "proj_xyz789",
  "name": "我的新專案",
  "created_at": "2026-03-15T08:30:00Z",
  "status": "active"
}

取得任務列表

支援分頁與篩選參數：

GET /v1/projects/:project_id/tasks?status=in_progress&limit=20&offset=0

支援的篩選參數包括 status（待辦/進行中/已完成）、assignee（指派對象 ID）、priority（high/medium/low）、due_before（截止日期前）。

速率限制

API 請求頻率限制如下：Free 方案每分鐘 60 次請求、Pro 方案每分鐘 600 次、Enterprise 方案每分鐘 6000 次。超過限制時將回傳 HTTP 429 狀態碼。
